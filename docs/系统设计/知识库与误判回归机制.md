# 知识库与误判回归机制

## 目标

系统不能只依赖用户逐条纠错。更稳妥的路线是建立可检查、可扩展的 GIS 知识库，让题目分析先依据结构化知识卡片和反例规则，再结合训练反思与 AI。

## 当前实现

- `autogis/knowledge_base.json`：结构化知识卡片。
- `autogis/knowledge_base.py`：根据题目匹配知识卡片，并识别反例规则。
- `/api/analyze-task`：分析结果中新增 `knowledge` 字段。
- 网页“AI 分析结果”中新增：
  - `知识库依据`：显示命中的操作类别、ArcGIS Pro 路径、QGIS 替代路径、检查点。
  - `反例排除`：显示为什么不应误判为某类任务。
- `tests/knowledge_regression.json`：固定误判回归样例。
- `python -m autogis knowledge-test`：运行知识库回归测试。

## 为什么不是直接“海量训练”

GIS 题目判断的核心不是堆大量文本，而是稳定区分相近概念：

- “影像”不一定是遥感。
- “分类”不一定是监督分类。
- “道路距离”不一定是网络分析。
- “山体阴影”不属于遥感处理。

如果直接抓取大量博客或教程，低质量内容会放大误判。当前方案优先使用结构化知识卡片和反例库，之后再逐步引入官方文档和可靠开源项目。

## 后续扩展

1. 扩展官方工具卡片：
   - ArcGIS Pro 工具路径
   - QGIS Processing 工具路径
   - GDAL / WhiteboxTools / OTB 适用场景

2. 增加题型卡片：
   - 适宜性评价
   - 分区统计
   - 可视域
   - 土地利用变化
   - 网络分析
   - 遥感指数与分类

3. 增加反例：
   - 日照 vs 山体阴影
   - 道路缓冲 vs 网络分析
   - 土地利用重分类 vs 遥感监督分类
   - 底图影像 vs 遥感处理

4. 扩展自动回归测试：
   - 每次改规则后，必须跑 `python -m autogis knowledge-test`。
   - 如果不该出现的类别出现，测试失败。

## 用户训练反思的作用

训练反思仍然有价值，但不应该承担全部判断压力。它应作为：

- 个性化竞赛经验库
- 错题复盘库
- 相似题案例库
- AI 提示词参考

系统的基础判断应来自知识库和反例库，训练反思用于提高贴合你的解题习惯。
