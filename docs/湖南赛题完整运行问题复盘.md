# 湖南赛题完整运行问题复盘

## 结论

这个工具不应该只停留在轻量表单。竞赛/科研型 GIS 处理需要由 Web 界面驱动本地 QGIS、GDAL、WhiteboxTools、Python 后台模块，并显示中间结果、输出路径、参考答案校验和失败原因。

## 已接入

- Web 按钮：`完整运行湖南赛题并校验`
- 后端接口：`POST /api/run-first-hunan`
- 后台模块：`autogis/competition_runner.py`
- 实际执行脚本：`scripts/run_first_hunan_full.py`
- 输出目录：`D:\桌面\AutoGIS结果\first_hunan_full_run`
- 报告：`D:\桌面\AutoGIS结果\first_hunan_full_run\run_report.md`

## 本次暴露的问题

1. 不能只说“跑通”。必须和参考答案或人工结果对比，否则用户无法判断结果是否可信。
2. WhiteboxTools 与 ArcGIS/QGIS 在部分水文算法、流向编码、汇流累积和河网提取上可能不完全等价。
3. `ExtractStreams` 和“汇流累积栅格 > 500”不是同一个操作，后续缓冲、擦除、候选森林、渔网点、视域面积会连锁偏差。
4. 多部分转单部分不能只依赖 Shapely 的 `MultiPolygon` 判断，ArcGIS/QGIS 对多环、多部件面拆分结果可能不同。
5. 中文路径、ArcInfo Grid、WhiteboxTools 组合不稳定，后台应统一先转换到 ASCII 工作目录，再把结果复制到用户指定输出目录。

## 当前参考答案校验

参考截图中空 5-22 已被写入校验表。当前匹配项包括 DEM 分辨率、填洼、流向、流量、阈值面积、森林原始面积和若干工具名称；不匹配项主要集中在河网提取、缓冲面积、单部件数量、候选区面积、候选点数量和视域面积。

## 下一步必须优化

1. 增加 ArcGIS/QGIS 等价算法模式：优先使用 QGIS/GRASS 工具复刻比赛流程，WhiteboxTools 作为备选引擎。
2. 对水文模块拆成明确步骤：填洼、流向、汇流累积、阈值栅格、栅格转线、缓冲、擦除。
3. 对每个步骤保存中间结果，并在界面显示“本步骤输出是否与参考答案一致”。
4. 增加答案表/历史答卷导入，让系统自动学习你的比赛结果并作为校验基准。
5. 把“完整运行湖南赛题”抽象成通用“竞赛题模板运行器”，支持多个题组、多个数据目录和多套参考答案。
